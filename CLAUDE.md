## How Inertia Works

Rails controllers render Vue components directly — no JSON API, no vue-router.

```ruby
render inertia: 'AcademicPeriods/Index', props: {
  academic_periods: periods.as_json
}
```

```vue
<script setup lang="ts">
defineProps<{ academic_periods: AcademicPeriod[] }>()
</script>
```

- Rails owns routing, sessions, CSRF, auth.
- `<Link>` component for navigation. `useForm()` for submissions. Errors flow back automatically.
- Layouts assigned per-page or globally via `createInertiaApp` resolve function.
- Auth pages override: `defineOptions({ layout: AuthLayout })`.

### CRITICAL: Inertia Form Params

**Inertia's `useForm()` sends flat params. Rails expects nested params.** Always use `.transform()` to wrap:

```typescript
// WRONG — will cause ActionController::ParameterMissing
form.post(academicPeriodsPath())

// CORRECT — wraps under the model key Rails expects
form.transform(data => ({ academic_period: data })).post(academicPeriodsPath())
```

This applies to ALL form submissions (`.post()`, `.patch()`, `.put()`). The wrapper key must match the Rails controller's `params.require(:model_name)`.

### Inertia Error Format

Inertia returns validation errors as `Record<string, string[]>` (arrays). Form components accept `error?: string | string[]` and display the first error via the `FormField` component.

## Design Patterns

### Rails

- **Inertia-first.** `render inertia:` not JSON. Only add API endpoints when genuinely needed.
- **PascalCase page paths.** `render inertia: 'AcademicPeriods/Index'` matches `app/frontend/pages/AcademicPeriods/Index.vue`.
- **Service objects** for complex operations (e.g., `AiTutorService`).
- **Scoping:** Always through `Current.organization` and `Current.membership`.
- **String-backed enums.** Never integer-backed. Use `enum :status, STATUSES.index_by(&:itself), validate: true`. Do NOT add a separate `validates :status, inclusion:` — the enum's `validate: true` handles it.
- **JSONB** for structured/flexible data (settings, schedule).
- **RESTful resources.** Skinny controllers, fat models/services.
- **Shallow route nesting.** Max 1 level deep. `resources :courses, shallow: true do ... end`.
- **Pessimistic locking** for capacity enforcement: `section.with_lock { ... }`.
- **Rails encrypted credentials** for API keys (`bin/rails credentials:edit`), never ENV vars for secrets.

### Authorization

- **Custom policy objects** in `app/policies/`. Base: `ApplicationPolicy.new(membership, record)`.
- **`authorize!`** in every controller action. Pass `policy_class:` explicitly.
- **`verify_authorized`** after_action in `InertiaController` — raises if no `authorize!` was called.
- **`skip_verify_authorized`** for actions without domain authorization (dashboard, settings, onboarding).
- **Permissions shared via Inertia** in `auth.can` hash. Frontend reads via `usePermissions()` composable.
- **Role hierarchy:** admin > tutor > tutorado.

### Vue

- **SFCs only:** `<script setup lang="ts">`. No Options API.
- **shadcn-vue first.** Always install via CLI (`bunx shadcn-vue@latest add <component>`) — never create custom alternatives. Use `nvm use 22` first if needed.
- **Composables:** `use*` convention (`usePermissions`, `useAiChat`, `useAppearance`).
- **State:** Inertia props for page data. Pinia only for global client state. Composables for shared logic.
- **Navigation:** `<Link>` or `router.visit()`. Never `<a href>` for internal links.
- **Route helpers:** Always import from `@/routes` (generated by js-routes). **Never hardcode URL strings** like `"/dashboard"` or `` `/sections/${id}/enrollments` ``. Use `dashboardPath()`, `sectionEnrollmentsPath(id)`, etc.
- **Forms:** `useForm()` with `.transform()` to nest params (see "CRITICAL" section above).
- **Breadcrumbs:** Last item renders as `BreadcrumbPage` (non-clickable). Middle items use route helpers.

### i18n

- `vue-i18n`. Default: `es`. Cookie: `app_lang`.
- One file per domain: `common`, `auth`, `nav`, `academic`, `enrollment`, `attendance`, `exercises`, `ai`, `dashboard`, `landing`, `onboarding`.
- Templates: `$t('key')`. Script: `const { t } = useI18n()`.
- Reactive options must be `computed()`. Always add both locales.

### Form Components

- Reusable components in `app/frontend/components/form/`: FormInput, FormSelect, FormTextarea, FormCheckboxGroup, FormRadioGroup, FormDatePicker, FormNumberInput, FormSwitch, SectionCard.
- Every form component accepts `error?: string | string[]` (Inertia returns arrays).
- Every form component works for both creating (empty props) and editing (populated props). One component, one UX.
- Built on shadcn-vue primitives with consistent error display and i18n support.

### LaTeX Rendering

- **MarkdownLatex component**: Uses `markdown-it` + `@mdit/plugin-katex` for AST-level `$...$` and `$$...$$` parsing. Output sanitized with DOMPurify before `v-html`.
- **LatexRenderer component**: Direct KaTeX rendering for standalone expressions.
- **ExerciseEditor component**: Split-pane editor (desktop) / tabbed (mobile) with live preview.
- **Never use regex** to parse LaTeX delimiters in markdown.

## Testing

### Backend (RSpec)

- Model: RSpec + Shoulda Matchers (`spec/models/`)

### Frontend (Vitest)

- Environment: `happy-dom` (not jsdom — jsdom has ESM compatibility issues)
- Unit: pure functions (`spec/frontend/unit/`)
- Component: Vue Test Utils (`spec/frontend/components/`)
- Mock Inertia: `vi.mock("@inertiajs/vue3", () => ({ usePage: vi.fn(), Link: { template: '<a><slot/></a>', props: ['href'] } }))`
- Mock vue-i18n: `vi.mock("vue-i18n", () => ({ useI18n: () => ({ t: (key: string) => key }) }))`
- Mock icons with `as any` cast: `const fakeIcon = { template: "<svg/>" } as any`
- Locale-agnostic assertions. `vi.useFakeTimers()` for dates.

## Common Pitfalls

1. **ParameterMissing on form submit** → Missing `.transform()` to nest params. See "CRITICAL" section.
2. **Multiple sidebar items highlight** → NavMain `isActive()` uses `startsWith` matching. Ensure unique hrefs.
3. **Double enum validation errors** → Don't add `validates :status, inclusion:` when using `enum validate: true`.
4. **Vitest crypto.hash error** → Running with Node 19. Use `nvm use 22`.
5. **Hardcoded routes break silently** → Always use route helpers from `@/routes`.
6. **AI message duplication** → `build_messages` queries DB history; don't append the current message again.
7. **Cross-section attendance injection** → Always validate enrollment_ids belong to the session's section.
